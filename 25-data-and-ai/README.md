# 25 — Data & AI Strategy: Build on Data, Scale with AI

> Data and AI are becoming core to every SaaS. This department covers data architecture, AI/ML feature strategy, data governance, AI cost management, and training data — so you build data infrastructure and AI features intentionally.

## Purpose

Most SaaS products are adding AI capabilities. This department helps you plan data pipelines, AI feature roadmap, governance, cost controls, and training data strategy — avoiding ad-hoc AI that burns budget and creates technical debt.

| Subfolder | What You'll Produce | Priority |
|---|---|---|
| [data-architecture](./data-architecture/) | Data pipeline, warehouse, ETL, models | P1 |
| [ai-features](./ai-features/) | AI/ML feature roadmap, build vs buy | P1 |
| [data-governance](./data-governance/) | Classification, access, retention, PII | P1 |
| [ai-costs](./ai-costs/) | Token/compute budgets, cost optimization | P1 |
| [training-data](./training-data/) | Data sourcing, labeling, synthetic, QA | P2 |

## How to Work Through This Department

1. **Data Architecture** — Plan pipelines, warehouse, models
2. **AI Features** — Roadmap AI capabilities; build vs buy
3. **Data Governance** — Classification, access, retention
4. **AI Costs** — Budget and optimize AI spend
5. **Training Data** — Sourcing, labeling, quality

## Time Estimate

| Approach | Time |
|---|---|
| Light (AI features + costs) | 1–2 weeks |
| Full (architecture + governance + training) | 3–4 weeks |

## Key Principle

**AI is a feature, not a strategy.** Start with clear use cases. Manage costs from day one. Governance before scale.
