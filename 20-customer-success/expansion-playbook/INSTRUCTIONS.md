# Expansion Playbook

## Overview

This folder is for **upsell and cross-sell motions** — when to suggest an upgrade, how to identify champions, and how to run expansion conversations. Expansion revenue (NRR) is a key driver of SaaS valuation.

## Questions to Answer

1. **What expansion opportunities exist?** (Tier upgrade, seats, add-ons, usage?)
2. **What triggers an expansion conversation?** (Usage limits, feature requests, QBR?)
3. **Who is the champion?** (Primary user, decision maker, budget holder?)
4. **What's your expansion pitch?** (Value prop, ROI story?)
5. **Do you offer expansion incentives?** (Annual prepay discount?)
6. **How do you track expansion pipeline?** (CRM opportunity?)
7. **Who owns expansion?** (CS, sales, or hybrid?)

## Output Template

Create a `README.md` in this folder with:

```markdown
# Expansion Playbook: [Your SaaS Name]

## Expansion Triggers
| Trigger | Signal | Action |
|---|---|---|
| Seat limit | 80%+ seats used | Offer seat add-on |
| Feature request | Request for higher-tier feature | Present upgrade |
| Usage spike | Sustained usage growth | Suggest scale-up |
| Contract renewal | 90 days out, healthy account | Annual + expand |
| QBR | Success + new use case | Propose add-on / tier |

## Champion Identification
- **Primary user:** [Role - day-to-day user]
- **Champion:** [Role - internal advocate]
- **Decision maker:** [Role - signatory]
- **Budget holder:** [Role - if different]
- **How to identify:** [Usage data, support tickets, QBR attendance]

## Expansion Pitch Framework
1. **Recap value delivered** — Usage, outcomes
2. **Introduce opportunity** — New tier, seats, add-on
3. **ROI / Business case** — Why it's worth it
4. **Offer** — Pricing, incentive (annual discount)
5. **Next step** — Proposal, trial, pilot

## Expansion by Type
| Type | Typical Discount | Owner |
|---|---|---|
| Seat add-on | 10-20% volume | CS |
| Tier upgrade | 10-15% | CS / Sales |
| Add-on product | Case-by-case | Sales |
| Annual prepay | 15-20% | CS / Sales |
```
