# Voice and Tone

## Overview

This folder is for **how you write** — voice (personality, consistent) and tone (adapts to context). Voice and tone guide every piece of copy: landing page, emails, support, in-app, and social.

## Questions to Answer

1. **What's your brand voice?** (3-5 adjectives: e.g., friendly, expert, concise)
2. **How do you adapt tone by channel?** (Support = empathetic, landing = confident?)
3. **What do you avoid?** (Jargon, slang, passive voice?)
4. **Do you use "we" or "you" focus?** (Customer-first)
5. **What's your sentence style?** (Short, punchy vs. flowing?)
6. **Do you have a style guide?** (Capitalization, spelling, punctuation?)
7. **Any examples of good/bad copy?** (Do/don't)

## Output Template

Create a `README.md` in this folder with:

```markdown

## Brand Voice
Our voice is:
- **[Adjective 1]:** [Definition, example]
- **[Adjective 2]:** [Definition, example]
- **[Adjective 3]:** [Definition, example]

## Tone by Context
| Context | Tone | Example |
|---|---|---|
| Landing page | Confident, benefit-driven | |
| Support | Empathetic, helpful, clear | |
| Error messages | Clear, apologetic, actionable | |
| Marketing email | Engaging, value-first | |
| In-app | Concise, task-focused | |
| Social | Casual, personable | |

## Writing Principles
- **Customer-first:** "You" over "we"
- **Clear over clever:** No jargon
- **Active voice:** "We send" not "It is sent"
- **Concise:** Short sentences, scannable
- **[Custom principle]:** 

## Do / Don't
| Do | Don't |
|---|---|
| Use plain language | Use jargon or buzzwords |
| Be helpful | Be passive-aggressive |
| Be specific | Be vague |
| Acknowledge feelings | Dismiss concerns |
```
